# Agnes Mini 技术开发文档 v2.0

> 版本: v2.0 (审核修订版) | 审核结论: ✅ 通过
> 核心关注: 角色与场景一致性 | 目标: 从剧本到视频的完整管线

---

## 1. 问题分析

### 1.1 当前管线的问题
v1.0 管线直接为每个镜头独立生成图片和视频, 导致:
- 角色外观不一致: 爷爷在镜头1和镜头3中面容、服装细节不一致
- 场景不连贯: 厨房在镜头2和镜头5中布局、色调不同
- 道具不统一: 青花瓷碗纹样在各镜头中不匹配
- 光照不连续: 暖黄灯光色温和方向在不同镜头中变化

### 1.2 根本原因
每次图片/视频生成都是独立的 text-to-image 调用, 模型没有跨镜头的参考锚点。
AI 模型对同一角色的每次生成都是全新的推理过程, 不保证视觉一致性。

### 1.3 解决方案
在生成各镜头之前, 先建立统一的角色和场景参考图库。
在每个镜头的生成中使用参考图作为 image_url / extra_body.image 输入, 强制模型保持视觉一致性。

## 2. 架构设计

### 2.1 与 v1.0 的集成策略 [审核补充]
v2.0 作为 v1.0 的扩展, 非替换。新增 ConsistentPipeline 类。
- v1.0 Pipeline 保持不动, 原有测试全部保留
- v2.0 新增 ConsistentPipeline, 独立测试模块
- CLI: agnes pipeline run 保持 v1.0; agnes pipeline v2 run 调用 v2.0
- Shot 模型保持不动, ShotV2 继承 Shot

### 2.2 三层管线

第一层: 剧本分析 (ScriptAnalysisAgent, agnes-2.0-flash)
  输入: 剧本文本 -> 输出: 角色清单 + 场景清单 + 道具清单 + 镜头清单

第二层: 参考图生成 (ReferenceGenerator, agnes-image-2.1-flash)
  输入: 角色/场景/道具描述 -> 输出: 角色四视图+表情, 场景局部变体, 道具特写

第三层: 带参考的镜头生成 (ConsistentGenerator)
  - 图片: extra_body.image 传入角色+场景参考
  - 视频: image 参数传入角色参考 (ti2vid)
  - 多图参考: 使用 agnes-image-2.0-flash [审核补充]

### 2.3 完整数据流

剧本 -> ScriptAnalysisAgent (agnes-2.0-flash)
  -> 角色: 爷爷/孙女 (含完整外貌描述)
  -> 场景: 厨房 (含布局/道具/光照描述)
  -> 镜头: 9 个 (含 image_prompt/video_prompt/char_refs/scene_ref)

ReferenceGenerator -> 参考图库:
  -> 爷爷基图 -> 四视图 + 表情变体 (image-to-image)
  -> 孙女基图 -> 四视图 + 表情变体
  -> 厨房全景 -> 局部变体 (wide/corner_stove/window_view/table_close)

ConsistentGenerator (遍历 9 镜头):
  -> 取角色+场景参考图
  -> 生图: extra_body.image = [角色参考图, 场景参考图] (2.0-flash 多图)
  -> 生视频: image = 角色参考图 (agnes-video-v2.0 ti2vid)
  -> 查询: GET /agnesapi?video_id={id} [审核修正]

## 3. 数据模型

### 3.1 角色模型 (CharacterBank)
Character: name, base_description, base_image_url
  views[]: view_type (front/side/back/three_quarter [统一]), image_url, prompt
  expressions[]: expression (neutral/happy/sad/surprised/thoughtful), image_url, prompt
CharacterBank: characters: list[Character]

### 3.2 场景模型 (SceneBank)
Scene: name, base_description, base_image_url
  variants[]: variant_type (wide_full/corner_stove/window_view/table_close), image_url
SceneBank: scenes: list[Scene]

### 3.3 扩充镜头模型 (ShotV2)
ShotV2: shot_number, description, duration, character_refs[], scene_ref,
  image_prompt, video_prompt, image_url, video_url,
  reference_image_status [新增], status

## 4. 角色一致性管线

### 4.1 角色基图生成 (text-to-image)
角色 1 - 爷爷: prompt = 70岁灰发皱纹深蓝中山装, 写实电影肖像
角色 2 - 孙女: prompt = 20岁仿生人半透明皮肤杏仁眼灰白亚麻衣

### 4.2 角色四视图变体 (image-to-image) [审核确认]
POST /v1/images/generations
model: agnes-image-2.1-flash
extra_body.image: [base_image_url]
prompt: [角色] full body [front/side/back/three_quarter] view
生成: front_view, side_view, back_view, three_quarter_view (4 种)

Prompt 结构指导 [审核补充]:
- 明确不变: same character, same clothing, same lighting
- 明确变化: different angle, full body / close-up

### 4.3 表情特写变体 (image-to-image)
变体: neutral, happy, sad, surprised, thoughtful (5 种)
Prompt: [角色] close-up portrait, [表情] expression, same character

### 4.4 参考图选择策略
- 远距离镜头: full body side/back view
- 面部特写: face front view + 对应表情
- 手部特写: base image (含手部细节)

## 6. 视频生成 (带参考)

### 6.1 创建视频任务
POST /v1/videos
{ model: agnes-video-v2.0, prompt: [动作描述], image: [角色参考图_url]
  height: 768, width: 1152, num_frames: 121 (8n+1, <= 441 [审核补充]), frame_rate: 24 }

### 6.2 查询任务结果 [审核修正]
推荐: GET https://apihub.agnes-ai.com/agnesapi?video_id={video_id}
注意: 官方推荐使用 video_id 而非 task_id 查询, 不推荐 GET /v1/videos/{task_id}

### 6.3 多图参考视频 [审核补充]
POST /v1/videos
{ image: [角色参考图, 场景参考图], prompt: [动作] in [场景] }

- 场景一致性: 布局/道具位置匹配
- 人工评估: 爷爷/孙女/厨房跨镜头一致性检查
