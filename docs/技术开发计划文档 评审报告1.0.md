一、需要修正的问题
⚠️ 问题 1: pytest 标记配置不完整
位置: 4.3 节

问题描述: pyproject.toml 配置中使用了 "not integration" 语法，但 markers 中未声明 integration 标记。

toml
[tool.pytest.ini_options]
markers = [
    "integration: marks tests that require real API key (deselect with '-m \"not integration\"')",
    "e2e: marks end-to-end tests (deselect with '-m \"not e2e\"')",
    "slow: marks tests that take more than 30 seconds (deselect with '-m \"not slow\"')",
]
修正: 配置本身是完整的（三个标记均已声明），但需注意：

-m "not integration" 在 pytest 中需要标记已注册才能正常跳过

如果开发者忘记注册标记，pytest 会发出警告但不会失败

建议补充:

toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["tests"]
addopts = "-v --strict-markers"   # ⬅️ 新增：严格模式，强制注册所有标记
markers = [...]

二、细节优化建议（非阻塞）
🔵 建议 1: 第 1 天"初次提交"之前应完成 .gitignore
当前第 1 天任务顺序：

初始化目录 → 2. pyproject.toml → 3. .gitignore → 4. .env.example → 5. LICENSE → 6. 安装依赖 → 7. 配置 ruff/mypy → 8. Git init → 9. 初次提交

优化: 建议 .gitignore 在 Git init 之前完成，确保首次提交不包含无关文件。

diff
+ | 编写 .gitignore | .gitignore | 15 min | 无 |
  | 初始化 Git 仓库 | — | 5 min | 全部上方 |
- | 初次提交 | — | 5 min | Git init |
🔵 建议 2: 第 9 天"关键帧动画模式测试"的验收标准不够具体
当前: "关键帧动画模式测试" — 20 min

建议补充:

text
- [ ] VideoProvider.create_task() 支持 extra_body.mode="keyframes"
- [ ] extra_body.image 接受列表 [url1, url2, ...]
- [ ] 返回的视频包含关键帧之间的平滑过渡
🔵 建议 3: 增加"每日站立会检查点"
当前计划是单人/小团队执行模型，没有团队同步机制。建议在每天任务末尾增加 10 分钟的同步检查点：

markdown
**每日结束检查**:
- [ ] 当日所有任务已完成
- [ ] 测试通过
- [ ] ruff/mypy 通过
- [ ] 代码已提交
对于 2 人以上团队，建议在每个 Phase 结束时增加 Phase Review 会议（30 分钟）。

🔵 建议 4: 视频生成测试的"慢测试"标记
视频生成测试（真实 API 调用）通常需要数十秒，已被标记为 @pytest.mark.slow。建议在 4.3 节的标记列表中，将 slow 的默认行为改为跳过，避免开发时意外触发长时间等待。

toml
[tool.pytest.ini_options]
addopts = "-v --strict-markers -m 'not slow'"

当前提交类型说明：

text
