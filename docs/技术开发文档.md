## 1. 项目概述

### 1.1 项目定位

Agnes Mini 是一个基于 **Agnes AI 免费 API** 的多模态 Agent 框架。它将 Agnes AI 提供的
文本推理、图像生成、视频生成三大能力整合为一个统一的 Agent 系统，让开发者通过
**单一 API Key** 即可构建多模态 AI 应用。

核心差异化：传统多模态 Agent 通常需要同时付费使用 OpenAI（文本）+ Midjourney（图像）
+ Runway（视频）等多个服务，而 Agnes Mini **全部使用 Agnes AI 免费 API**，
显著降低开发和运营成本。

### 1.2 核心能力

| 能力 | 使用模型 | 说明 |
|------|---------|------|
| **文本生成与推理** | agnes-2.0-flash | 对话、代码、推理、Agent 规划、工具调用 |
| **图像生成与编辑** | agnes-image-2.1-flash | 文生图、图生图、多图合成，支持 1K~4K 分辨率 |
| **视频生成** | agnes-video-v2.0 | 文生视频、图生视频、关键帧动画（异步 API） |
| **多模态理解** | agnes-2.0-flash | 图像理解、截图分析、视觉问答（通过 image_url） |
| **Agent 工作流** | agnes-2.0-flash | 任务分解、工具调用、函数调用、计划执行 |

### 1.3 目标用户

- 想要构建 AI 应用但预算有限的独立开发者
- 需要快速原型验证的创业团队
- AI 教育场景和学生项目
- 需要多模态能力的 Agent 应用
- 希望在单一平台统一管理 AI 能力的团队

### 2.2 与 xyq 项目的架构对比

| 维度 | xyq（小云雀视频 Agent） | Agnes Mini |
|------|-----------------------|------------|
| LLM | OpenAI / Claude（需单独付费） | Agnes 2.0 Flash（免费） |
| 图像能力 | 无 | Agnes Image 2.1 Flash（免费） |
| 视频能力 | 小云雀 API | Agnes Video V2.0（免费） |
| API Key 数量 | 至少 2 个（LLM + 视频） | 1 个（全部统一） |
| Provider 抽象 | 仅视频 Provider | 文本 + 图像 + 视频 三层 Provider |
| 核心定位 | 单一视频生成工具 | 通用多模态 Agent 框架 |

### 2.3 核心设计原则

1. **统一 API 层**: 所有模型共用 Base URL https://apihub.agnes-ai.com/v1，单一认证机制
2. **Provider 抽象**: 每一类模型（文本/图像/视频）都通过抽象接口封装，便于未来切换后端
3. **Agent 编排**: 通过 Orchestrator Agent 统一调度各子 Agent，支持复杂多步骤工作流
4. **全免费**: 核心模型当前全部免费，框架本身不引入任何付费依赖
5. **异步优先**: 所有 I/O 操作基于 asyncio
6. **可测试**: 每一层 Provider 都有对应的 Mock 实现，支持无 API Key 开发测试

### 2.4 核心数据流

用户输入（自然语言）
    |
    v
OrchestratorAgent.run(user_input)
    |
    +--- 意图识别与任务规划（合并为一次 LLM 调用，减少往返开销）
    |    System Prompt 指示模型同时输出：
    |    - 意图类型（文本/图像/视频/组合）
    |    - 有序子任务列表
    |
    +--- 子任务执行
    |    +--- 文本任务 -> TextAgent -> ChatProvider
    |    +--- 图像任务 -> ImageAgent -> ImageProvider
    |    +--- 视频任务 -> VideoAgent -> VideoProvider（异步轮询）
    |
    +--- 结果汇总（使用 agnes-2.0-flash）
         整合各子任务结果，生成最终回复

最终输出：文本回复 / 图像 URL / 视频 URL / 组合结果

## 4. 模块详解

### 4.1 配置管理（src/config.py）

基于 Pydantic Settings v2，通过 .env 文件和系统环境变量管理所有配置。

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| agnes_api_key | str | "" | Agnes AI API Key。为空时系统行为见下方说明 |
| agnes_base_url | str | https://apihub.agnes-ai.com/v1 | API Base URL |
| text_model | str | agnes-2.0-flash | 文本模型名称 |
| image_model | str | agnes-image-2.1-flash | 图像模型名称 |
| video_model | str | agnes-video-v2.0 | 视频模型名称 |
| default_temperature | float | 0.7 | 默认生成温度 |
| default_max_tokens | int | 4096 | 默认最大输出 tokens |
| video_poll_interval | float | 2.0 | 视频轮询初始间隔（秒）|
| video_max_interval | float | 30.0 | 视频轮询最大间隔（秒）|
| video_timeout | int | 600 | 视频生成总超时（秒）|
| api_retry_count | int | 3 | 单次 API 请求重试次数 |
| api_retry_backoff | float | 1.0 | 重试退避系数（秒），每次翻倍 |
| log_level | str | INFO | 日志级别 |
| output_dir | str | ./output | 输出目录 |

关键设计：
- **单例模式**: get_settings() 返回全局唯一 Settings 实例
- **热重载**: reload_settings() 支持运行时重新加载配置
- **环境变量优先级**: 系统环境变量 > .env 文件 > 默认值
- **API Key 为空时的行为**: 如果 agnes_api_key 为空字符串，系统自动回退到 Mock Provider 模式运行。这允许开发者在没有 API Key 的情况下进行开发和测试。生产环境中应抛出 ConfigError 阻止启动。
- **重试机制**: 所有 Provider 层的 HTTP 请求均支持自动重试，最多 api_retry_count 次，退避间隔按 api_retry_backoff 翻倍递增。

### 4.5 Provider 层（src/providers/）

#### 抽象基类（base.py）

class AgnesProvider(ABC)
    base_url, api_key
    def _headers() -> dict
    def _url(path) -> str
    async def _request_with_retry(method, url, **kwargs) -> Response
        # 自动重试：最多 api_retry_count 次，退避间隔翻倍
        # 对 429/502/503/504 等可重试状态码进行自动重试
        # 对 400/401/403/422 等不可重试状态码直接抛出异常

#### ChatProvider（chat.py）

chat(request) — 标准对话生成
chat_stream(request) — 流式对话
chat_with_tools(request, tools) — 工具调用
chat_with_images(messages, images) — 多模态图像理解
chat_with_thinking(request, budget) — Thinking 深度推理

端点: POST /v1/chat/completions

#### ImageProvider（image.py）

text_to_image(request) — 文生图
image_to_image(prompt, images, size, ratio) — 图生图
multi_image_composition(prompt, images, size, ratio) — 多图合成
download_image(url, output_path) — 下载图片到本地

端点: POST /v1/images/generations

注意事项:
- response_format 必须放在 extra_body 内，不在顶层
- 图生图不需要传 tags: ["img2img"]
- 图片 URL 必须可公开访问，否则使用 Data URI Base64
  格式示例: data:image/png;base64,iVBORw0KGgo...
  注意：大尺寸图片的 Base64 编码会显著增加请求体大小，建议优先使用公网 URL
- 推荐客户端超时 60s-360s

#### VideoProvider（video.py）

create_task(request) — 创建视频生成任务
query_task(video_id) — 查询任务状态
poll_task(video_id, timeout) — 轮询等待完成
download_video(url, output_path) — 下载视频到本地

创建端点: POST /v1/videos
查询端点（推荐）: GET /agnesapi?video_id=<VIDEO_ID>
查询端点（兼容）: GET /v1/videos/<TASK_ID>

注意：video_id 和 task_id 的关系
- create_task 返回的响应中包含 task_id 和 video_id 两个字段
- video_id 是推荐的查询方式，更稳定（使用 GET /agnesapi）
- task_id 仅用于兼容旧版集成（使用 GET /v1/videos/{task_id}）
- 新开发的应用应优先使用 video_id

视频轮询策略：
第 1 次 -> 2 秒
第 2 次 -> 5 秒
第 3 次 -> 10 秒
第 4+ 次 -> 30 秒（上限）
总超时 -> 600 秒（10 分钟）

轮询失败处理：
- 如果 query_task 返回 status == "failed"，立即停止轮询并抛出
  VideoPollingError，携带 API 返回的 error 信息
- 如果单次 HTTP 请求因网络抖动失败（502/503/504 等），使用内置的
  _request_with_retry 机制自动重试，最多 3 次
- 如果重试耗尽仍失败，抛出 APIError
- 如果总等待时间超过 video_timeout，抛出 VideoTimeoutError

#### OrchestratorAgent — 核心编排模块

职责:
1. 意图识别与任务规划（合并为一次 LLM 调用）:
   - 使用 System Prompt 指示 agnes-2.0-flash 同时输出：
     - 意图类型（文本/图像/视频/组合）
     - 有序子任务列表（含每个任务的输入和依赖关系）
   - 合并为一次调用可减少 token 消耗约 30-50%，降低延迟
2. 执行调度: 按顺序或依赖关系执行各子 Agent
3. 上下文传递: 使用标准化的 Context 对象在子任务间传递数据
4. 结果汇总: 整合各子任务输出，生成最终回复

上下文传递格式:

# 标准化的上下文对象
context = {
    "last_text_output": "",                    # 上一步的文本结果
    "last_image_url": None,                    # 上一步的图片 URL
    "last_image_path": None,                   # 上一步的本地图片路径
    "last_video_url": None,                    # 上一步的视频 URL
    "last_video_path": None,                   # 上一步的本地视频路径
    "conversation_history": [],                # 对话历史列表
    "user_preferences": {                      # 用户偏好（保留跨步骤）
        "style": None, "ratio": None
    },
    "shared_files": {}                         # 共享文件映射 {名称: 路径}
}

每个 Agent 执行完毕后，将结果写入 context；下一个 Agent 从 context 读取输入。

### 5.2 文本模型 API（agnes-2.0-flash）

端点: POST /v1/chat/completions

关键特性:

- 最大输出: 65.5K tokens

- 支持 Streaming
- 支持 Thinking 模式（两套兼容格式）
- 当前价格: 免费（标准价格请参考 Agnes AI 官方定价页面）

Thinking 模式:
OpenAI 兼容: { "chat_template_kwargs": { "enable_thinking": true } }
Anthropic 兼容: { "thinking": { "type": "enabled", "budget_tokens": 2048 } }

多模态图像理解:
messages: [{
  role: "user",
  content: [
    { type: "text", text: "描述这张图片" },
    { type: "image_url", image_url: { url: "https://..." } }
  ]
}]

### 5.3 图像模型 API（agnes-image-2.1-flash）

端点: POST /v1/images/generations

分辨率参考:

| 宽高比 | 1K | 2K | 3K | 4K |
|--------|------|------|------|------|
| 1:1 | 1024x1024 | 2048x2048 | 3072x3072 | 4096x4096 |
| 16:9 | 1312x736 | 2624x1472 | 3936x2208 | 5248x2944 |
| 9:16 | 736x1312 | 1472x2624 | 2208x3936 | 2944x5248 |
| 4:3 | 1152x864 | 2304x1728 | 3456x2592 | 4608x3456 |

支持文生图、图生图，推荐客户端超时 60s-360s。

注意事项:
- response_format 必须放在 extra_body 内，不在顶层
- 图生图不需要传 tags: ["img2img"]
- 图片 URL 必须可公开访问，否则使用 Data URI Base64
  格式示例: data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAE...
  超过 1MB 的 Base64 建议优先转为公网 URL 传递
- return_base64 仅按需开启，高分辨率 Base64 可能内存压力大
- 当前价格: 免费（标准价格请参考 Agnes AI 官方定价页面）

### 5.4 视频模型 API（agnes-video-v2.0）

创建任务: POST /v1/videos
查询结果（推荐）: GET /agnesapi?video_id=<VIDEO_ID>
查询结果（兼容）: GET /v1/videos/<TASK_ID>

帧数计算公式: num_frames = 8n + 1，其中 n 为正整数
num_frames 上限: 441
frame_rate 范围: 1-60
分辨率标准化: 480p / 720p / 1080p
当前价格: 免费（标准价格请参考 Agnes AI 官方定价页面）

常用时长参数:
~3秒: num_frames=81 = 8*10+1, frame_rate=24
~5秒: num_frames=121 = 8*15+1, frame_rate=24
~10秒: num_frames=241 = 8*30+1, frame_rate=24
~18秒: num_frames=441 = 8*55+1, frame_rate=24

视频时长计算: seconds = num_frames / frame_rate

任务状态: queued -> in_progress -> completed / failed

关键帧动画:
通过 extra_body.mode: "keyframes" 和 extra_body.image: [url1, url2] 实现。
系统自动在关键帧之间生成平滑过渡动画。

依赖管理说明:
- pyproject.toml: 项目的主依赖声明文件，包含所有顶层依赖及版本范围。
  开发者在此文件中添加和更新依赖。
- requirements.txt: 仅用于 CI/CD 和部署环境的依赖锁定导出，
  应由 poetry export 或 uv export 自动生成，开发者不应手动编辑此文件。
- lock 文件 (poetry.lock / uv.lock): 用于锁定传递依赖的具体版本，
  确保开发、测试、生产环境依赖一致。应提交到版本控制。
- CI 中定期运行 pip audit / safety 检查依赖漏洞。
