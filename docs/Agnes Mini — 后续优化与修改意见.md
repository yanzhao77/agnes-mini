问题描述
当前文档中，各模型的价格标注方式不一致：

模型	章节价格标注	附录 A
agnes-2.0-flash	免费（标准价 Input $0.03/1M, Output $0.15/1M）	免费
agnes-image-2.1-flash	未标注标准价	免费
agnes-video-v2.0	免费（标准价 $0.005/秒）	免费
附录 A 中仅标注"免费"，各章节的详细程度不统一，且图像模型的标准价信息缺失。

优化建议
方案一（推荐）：统一为简洁模式

各章节统一标注为：

markdown
当前价格: 免费（标准价格请参考 Agnes AI 官方定价页面）
附录 A 同样标注为 免费，避免文档中维护价格信息的负担。

方案二：完整补齐

向 Agnes AI 官方确认图像模型的标准单价，并在 5.3 节补充：

markdown
当前价格: 免费（标准价 $0.04/张）
各章节统一为 "免费（标准价 $X/单位）" 格式。

建议采用方案一，原因：

API 价格可能变动，文档维护成本高

减少因价格信息过期导致的误导风险

将详细价格引导至官方渠道，更可靠

建议 C：明确 requirements.txt 与 pyproject.toml 的定位关系
属性	内容
优先级	🔵 低
影响范围	项目结构 / 依赖管理
建议类型	工程规范
预计工作量	15 分钟
问题描述
文档项目结构中同时包含了 pyproject.toml 和 requirements.txt：

text
+--- pyproject.toml                  # 项目元数据
+--- requirements.txt                # 依赖列表
但同时文档又在多处建议使用 poetry/uv 管理依赖并生成 lock 文件。这两个文件同时存在可能导致：

依赖版本声明在两个文件中不同步

开发者不清楚应该用哪个作为依赖来源

维护两套依赖列表的额外负担

优化建议
方案一（推荐）：明确职责分工

在文档中增加说明，明确定义两个文件的用途：

markdown
依赖管理说明：
- pyproject.toml: 项目的主依赖声明文件，包含所有顶层依赖及版本范围
- requirements.txt: 仅用于 CI/CD 和部署环境的依赖锁定导出（由 poetry/uv 自动生成），
  开发者不应手动编辑此文件
并在项目结构注释中标注 requirements.txt 为 （由 poetry/uv 自动生成，请勿手动编辑）。

方案二：移除 requirements.txt

完全使用 pyproject.toml + lock 文件方案，移除 requirements.txt，避免混淆。

技术开发文档.md 第 3 节 项目结构 — 文件注释

技术开发文档.md 第 9 节 依赖关系图 — 补充说明

如采用方案二，在后续实际操作中移除 requirements.txt
